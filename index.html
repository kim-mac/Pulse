New Query Monitoring

  • What it does: Monitors a company, industry, or trend across news, hiring, sentiment, regulatory pressure, and competitor movement.
  • When to use it: Before interviews, market scans, competitive reviews, strategy work, or fast-moving research tasks.
  • What you get: A live agent dashboard and one intelligence brief with Urgent, Notable, FYI, recommended actions, and monitoring cadence.

Interview Prep

  • What it does: Builds a company-specific interview prep brief using the same monitoring engine, but tuned for role expectations and interview signals.
  • When to use it: Before interviews when you need fast context on company changes, candidate themes, and role-specific prep direction.
  • What you get: A prep brief with rounds, confidence, prep plan, warnings, and supporting notes.

CSV Analyst

  • What it does: Analyzes a single uploaded CSV for quality, schema, trends, anomalies, and key storylines.
  • When to use it: When you want a fast internal-data read before a meeting, project review, or decision.
  • What you get: A structured analyst-style brief with charts, quality scoring, and anomaly detection.

Compare Two Files

  • What it does: Compares two CSVs side by side to highlight quality differences, metric shifts, schema drift, and the most important change.
  • When to use it: When you need a before-vs-after snapshot across two datasets.
  • What you get: A comparison brief with a top key insight, file-overview cards, metric changes, and difference summaries.

Cross-Reference Internal and External Signals

  • What it does: Combines uploaded CSV findings with live external monitoring signals so you can see what your internal data means in the context of the market.
  • When to use it: When you want to connect internal sales, ops, customer, or performance data to competitor movement, sentiment, hiring patterns, or broader market changes.
  • What you get: A synthesized view of internal signal, external signal, and the combined takeaway so the dataset becomes more decision-ready than a standalone CSV read.

Recent Monitoring Runs

  • What it does: Saves completed monitoring runs locally in the browser so you can reopen a prior snapshot or rerun the same topic.
  • When to use it: When you want to track a topic over time instead of treating each run as a one-off scan.
  • What you get: Saved runs, re-run shortcuts, and a “What Changed Since Last Run” summary when PulseBoard finds a prior comparable run.

Provider Setup

  • What it does: Lets you connect OpenAI, Anthropic, Gemini, or NVIDIA for the current tab session.
  • When to use it: Before any live monitoring or CSV analysis.
  • What you get: A session-only connection that clears on refresh and powers the rest of the app without permanently storing credentials.

Where to get each provider key

OpenAI Open Console

Create an OpenAI secret key

Best fit if you want PulseBoard monitoring and CSV analysis through models like gpt-5.4.

Paid API usage
  1. Sign in to the OpenAI platform and open the API keys page.
  2. Create a new secret key for the project or organization you want to use.
  3. Copy it once and paste it here with the default OpenAI endpoint and a model your OpenAI Platform project can access.
Claude / Anthropic Open Console

Create an Anthropic API key

Use this if you want the relay to run PulseBoard through Claude models like claude-sonnet-4-20250514 and the Anthropic Messages API.

Usually paid, small trial credits for new users
  1. Sign in to the Anthropic Console and open the keys settings area.
  2. Create a key for your workspace, then copy it before leaving the dialog.
  3. Paste it here and keep the default Anthropic endpoint unless you use a custom relay route.
Google Gemini Open AI Studio

Create a Gemini API key

Google AI Studio is the fastest official route for generating a Gemini key for prototyping and internal tools.

Free tier available, paid tier optional
  1. Open AI Studio and sign in with the Google account tied to your project.
  2. Choose Get API key and generate a key for the project you want to bill.
  3. Copy the key into PulseBoard and keep the suggested Gemini model unless you intentionally want another one.
RecommendedNVIDIA API Open Build

Create an NVIDIA Build key

PulseBoard treats NVIDIA as an OpenAI-compatible endpoint, so you can use a Build key plus the NVIDIA base URL.

Build currently offers free inference access
  1. Sign in to NVIDIA Build and open your account, then click More models.
  2. Search OpenAI, select any model such as gpt-oss-120b, click View Code, generate a key, and copy the token that starts with nvapi-.
  3. Paste it here and leave the endpoint as https://integrate.api.nvidia.com/v1 unless you have a custom route.
